Processos JHipster

JHipster é uma ferramenta pra desenvolvimento de aplicações web. A plataforma implementa um gerador de código Yeoman pra criar aplicações baseadas na arquitetura Spring Boot + AngularJS.

Esse framework será utilizado nesse disciplina para exemplificar (e praticar) alguns processos de desenvolvimento de software. Escolhemos um conjunto de processos que, de forma simplificada, abordam os principais fluxos de desenvolvimento de uma aplicação web.

Serão cinco os processos que iremos utilizar:

  • Configurando o ambiente de desenvolvimento
  • Criando uma aplicação
  • Criando um CRUD simples
  • Criando um CRUD complexo
  • Alterando um CRUD

Configurando o ambiente

Esse processo os setup inicial do ambiente de desenvolvimento. Para realizar esse processo é necessário realizar as seguintes atividades:

  1. Instale o Java 8.
  2. Instale o Maven 3.x
  3. Instale o client Git
  4. Instale o client Git SourceTree
  5. Instale o Node.js
  6. Instale o Yeoman: npm install -g yo
  7. Instale o Bower: npm install -g bower
  8. Instale o Gulp: npm install -g gulp
  9. Instale o JHipster: npm install -g generator-jhipster

Criando uma aplicação (ou alterando as propriedades de uma aplicação)

Esse processo é usado para criar uma nova aplicação ou alterar algumas propriedades da aplicação. Normalmente é executado no início do processo de desenvolvimento e envolve definir a arquitetura da aplicação. Para realizar esse processo é necessário realizar as seguintes atividades:

  1. Se for uma nova aplicação, criar o diretório da aplicação mkdir myApp && cd myApp
  2. Iniciar o gerador yo jhipster. O gerador irá fazer 15 perguntas (listadas abaixo):
  3. Forneça valores apropriados para as 15 perguntas (quando se aplicar, usar o valor default sugerido pelo gerador)
  4. Executar o comando mvn spring-boot:run
  5. Executar o comando gulp

Criando um CRUD simples

Esse processo é usado para criar um CRUD simples. Um CRUD é considerado simples quando o seu desenvolvimento pode ser feito a partir do próprio gerador yo (sem uso de uma modelagem de entidades). Normalmente os CRUDs dessa categoria são de Entidades com poucos atributos e associações. Para realizar esse processo é necessário realizar as seguintes atividades:

  1. Sincronize o repositório de código local (faça o update e comite qualquer alteração)
  2. Execute o comando yo jhipster:entity <NOME_ENTIDADE> --[options]
  3. Para cada atributo, forneça as seguintes informações: nome, tipo e regras para validação.
  4. Para cada associação, forneça as seguintes informações: entidade associada, nome e tipo da associação.
  5. O gerador irá fazer mais três perguntas , forneça a resposta default sugerida pelo gerador para as três.
  6. Resolva os conflitos se existirem (indicando se deseja sobrescrever ou manter o código atual)
  7. Reverta algumas alterações feitas pelo gerador (aquelas que sobrescrevem as alterações feitas pelo programador)
  8. Implemente as customizações necessárias (tradução de mensagens, regras de negócio, código de teste, etc)
  9. Faça o update do código e resolva os conflitos se existirem
  10. Valide os testes com o código integrado
  11. Comite as alterações

Criando um CRUD complexo

Esse processo é usado para criar um CRUD complexo. Um CRUD é considerado complexo quando o seu desenvolvimento requer o uso de uma ferramenta de modelagem. Normalmente os CRUDs dessa categoria são de Entidades com vários atributos e associações. Para realizar esse processo é necessário realizar as seguintes atividades:

  1. Desenhe o modelo no JDL Studio
  2. Faça o download do arquivo no diretório .jhipster
  3. Sincronize o repositório de código local (faça o update e comite qualquer alteração)
  4. Execute o comando yo jhipster:import-jdl .jhipster/<NOMEDOARQUIVO>
  5. Resolva os conflitos se existirem (indicando se deseja sobrescrever ou manter o código atual)
  6. Reverta algumas alterações feitas pelo gerador (aquelas que sobrescrevem as alterações feitas pelo programador)
  7. Implemente as customizações necessárias (tradução de mensagens, regras de negócio, código de teste, etc)
  8. Faça o update do código e resolva os conflitos se existirem
  9. Valide os testes com o código integrado
  10. Comite as alterações

Alterando um CRUD

Esse processo é usado para alterar um CRUD (simples ou complexo). Para realizar esse processo é necessário realizar as seguintes atividades:

  1. Altere o arquivo .jhipster/<NOME_ENTIDADE>.json
  2. Sincronize o repositório de código local (faça o update e comite qualquer alteração)
  3. Execute o comando yo jhipster:entity <NOME_ENTIDADE> --regenerate
  4. Resolva os conflitos se existirem (indicando se deseja sobrescrever ou manter o código atual)
  5. Reverta algumas alterações feitas pelo gerador (aquelas que sobrescrevem as alterações feitas pelo programador)
  6. Implemente as customizações necessárias (tradução de mensagens, regras de negócio, código de teste, etc)
  7. Faça o update do código e resolva os conflitos se existirem
  8. Valide os testes com o código integrado
  9. Comite as alterações
